# Implementation Plan: Template Approval Workflow

**Branch**: `028-approval-workflow` | **Date**: 2026-05-24 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `formcraft-specs/specs/028-approval-workflow/spec.md`

## Summary

Implement template approval workflow with submit/review/approve/reject/publish lifecycle, element-level review comments, review queue with governance dashboard, org-level workflow toggle, and full audit trail. Extends existing backend state machine and template_reviews table — primarily a frontend + backend extension feature.

## Technical Context

**Language/Version**: Python 3.12 (backend), TypeScript / Angular 19 (frontend)
**Primary Dependencies**: FastAPI, Supabase (PostgreSQL + Auth), Angular Material, WeasyPrint (timeline PDF export)
**Storage**: Supabase PostgreSQL — extends existing `template_reviews` table, adds `department_default_reviewers` table
**Testing**: Manual testing per quickstart.md scenarios
**Target Platform**: Web application (desktop browsers)
**Project Type**: Web service (backend API + frontend SPA)
**Performance Goals**: Review queue loads < 2s for 500 templates, transitions < 1s
**Constraints**: Builds on existing state machine in `template_service.py`, no new roles needed
**Scale/Scope**: ~5 new backend endpoints, ~5 frontend components, 1 migration, extends 2 existing endpoints
