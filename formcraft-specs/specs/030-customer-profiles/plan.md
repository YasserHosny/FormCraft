# Implementation Plan: Customer Profiles & Auto-Populate

**Branch**: `030-customer-profiles` | **Date**: 2026-05-25 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `formcraft-specs/specs/030-customer-profiles/spec.md`

## Summary

Implement a customer profile management system integrated into Form Desk. Operators maintain an address book of known customers/entities with Arabic+English names, government identifiers, and contact info. During form filling, operators select a customer to auto-populate matching fields — reducing fill time by 40%+ for repeat customers. Features include customer CRUD with search, auto-populate with convention-based field mapping (plus designer-configurable overrides), cross-template submission history per customer, auto-create profiles from submissions, admin merge/deactivate/delete, org-configurable custom fields, and comprehensive audit logging for all customer data access.

## Technical Context

**Language/Version**: Python 3.12 (backend), TypeScript / Angular 19 (frontend)
**Primary Dependencies**: FastAPI, Supabase (PostgreSQL + Auth), Angular Material
**Storage**: Supabase PostgreSQL — new `customers` table, new `customer_field_mappings` table, extends `form_submissions` with `customer_id`, extends org settings with custom field schema
**Testing**: Manual testing per quickstart.md scenarios
**Target Platform**: Web application (desktop browsers)
**Project Type**: Web service (backend API + frontend SPA)
**Performance Goals**: Customer search <1s for 50k records, auto-populate <500ms, customer list page load <1s
**Constraints**: Extends existing Form Desk filler (F17), uses existing audit logger (F08), org-scoped via RLS (F25)
**Scale/Scope**: ~8 new backend endpoints, ~5 frontend components, 1 migration, extends 2 existing services (submission, org settings)
