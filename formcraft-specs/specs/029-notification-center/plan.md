# Implementation Plan: Notification Center

**Branch**: `029-notification-center` | **Date**: 2026-05-25 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `formcraft-specs/specs/029-notification-center/spec.md`

## Summary

Implement a system-wide notification engine with in-app notifications (bell icon, dropdown panel, full history page), email delivery (bilingual HTML with org branding), per-user notification preferences, and admin system announcements. Hooks into existing template transitions (F28), feedback replies (F14), and template versioning (F19) as event sources. Creates a new generalized `notifications` table replacing the feedback-specific `feedback_notifications` table for unified notification routing.

## Technical Context

**Language/Version**: Python 3.12 (backend), TypeScript / Angular 19 (frontend)
**Primary Dependencies**: FastAPI, Supabase (PostgreSQL + Auth), Angular Material, python-email (or equivalent SMTP library for email delivery)
**Storage**: Supabase PostgreSQL — new `notifications` table, new `notification_preferences` table, extends org settings
**Testing**: Manual testing per quickstart.md scenarios
**Target Platform**: Web application (desktop browsers)
**Project Type**: Web service (backend API + frontend SPA)
**Performance Goals**: In-app notifications within 5s, email queued within 1s, bulk create (500 recipients) within 3s
**Constraints**: Extends existing F14 notification endpoints, hooks into F28 transition_status(), polling-based (no WebSocket for MVP)
**Scale/Scope**: ~6 new backend endpoints, ~4 frontend components, 1 migration, extends 1 existing service
