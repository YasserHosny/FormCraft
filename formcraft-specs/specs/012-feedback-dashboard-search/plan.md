# Implementation Plan: Feedback Dashboard Search & Labels

**Branch**: `012-feedback-dashboard-search` | **Date**: 2026-05-07 | **Spec**: [spec.md](spec.md)  
**Depends on**: Feature 011 (`001-customer-feedback`) тАФ `feedback_submissions` table and admin dashboard must exist

## Summary

Extend the FormCraft admin feedback dashboard with debounced keyword search, multi-facet filtering (status, submitter, date range, labels), and a label management system. Admins create and assign named semantic-colour labels to submissions via an inline modal. Label filtering uses OR logic within the label facet and AND logic across facets.

## Technical Context

**Language/Version**: Python 3.12 (backend), TypeScript / Angular 17 (frontend)  
**Primary Dependencies**: FastAPI, Supabase (PostgreSQL), Angular Material  
**Testing**: pytest (backend unit + integration), Angular TestBed (frontend)  
**Target Platform**: Web (server: Linux, client: modern browsers)  
**Project Type**: Web application тАФ backend API extension + Angular SPA extension  
**Performance Goals**: Search results within 2 seconds at 10,000 submissions; filter updates within 1 second  
**Constraints**: Max 5 labels per submission (service-enforced); label names unique (DB UNIQUE constraint); colour stored as enum string; filter state session-only  
**Scale/Scope**: Single-tenant; ADMIN role only for all label and search endpoints

## Implementation Phases

### Phase 1 тАФ Backend: Migration, Schemas, Service Extension

1. Write `009_create_feedback_labels.sql` тАФ full SQL from data-model.md (tables, indexes, RLS)
2. Write `app/schemas/label.py`:
   - `LabelColour` (str enum: red, orange, yellow, green, teal, blue, purple, pink, grey, brown)
   - `LabelCreateRequest`, `LabelUpdateRequest`, `LabelResponse`
   - `SubmissionLabelAssignRequest` (label_ids: list[UUID], max 5)
   - Extend `FeedbackAdminItem` from feature 011 schemas to include `labels: list[LabelResponse]`
3. Extend `app/services/feedback/service.py`:
   - `list_feedback()`: add `search` (ILIKE) and `label_ids` (OR filter) parameters; JOIN labels in query
   - `list_submitters()` тЖТ distinct user_ids from `feedback_submissions` JOINed to `profiles` for display_name; ordered alphabetically
   - `create_label(admin_id, payload)` тЖТ insert + return
   - `update_label(id, payload)` тЖТ patch + return; 404 if not found; 409 on name conflict
   - `delete_label(id)` тЖТ cascade delete via FK; 404 if not found
   - `assign_labels(feedback_id, label_ids)` тЖТ enforce тЙд5 limit, replace full set (DELETE + INSERT)
4. Extend `app/api/routes/admin.py`:
   - `GET /api/admin/feedback` тАФ add `search` and `label_ids` query params
   - `GET /api/admin/feedback/submitters` тАФ distinct submitter list for autocomplete
   - `GET /api/admin/labels` тАФ list all labels
   - `POST /api/admin/labels` тАФ create label; 409 on duplicate name
   - `PATCH /api/admin/labels/{id}` тАФ update label
   - `DELETE /api/admin/labels/{id}` тАФ delete label
   - `PUT /api/admin/feedback/{id}/labels` тАФ replace label set on a submission

### Phase 3 тАФ Frontend

2. Build `LabelManagerComponent` (MatDialog modal):
