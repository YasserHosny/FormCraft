# Implementation Plan: Feedback Rich Media

**Branch**: `013-feedback-rich-media` | **Date**: 2026-05-07 | **Spec**: [spec.md](spec.md)  
**Depends on**: Feature 011 (`001-customer-feedback`) ‚Äî `feedback_submissions` table, single-image and audio upload endpoints, and submission widget must exist

## Summary

Extend the FormCraft feedback widget to support multiple image attachments (up to 5) and optional video recording/upload. The existing `image_url` column on `feedback_submissions` is replaced by a normalised `feedback_images` table; a new `video_url` column is added. The audio/video mutual-exclusion constraint is enforced both client-side and at the Pydantic layer. All upload/delete endpoints follow the two-step pattern established in feature 011.

## Technical Context

**Language/Version**: Python 3.12 (backend), TypeScript / Angular 17 (frontend)  
**Primary Dependencies**: FastAPI, Supabase (PostgreSQL + Storage), Angular Material  
**Testing**: pytest (backend unit + integration), Angular TestBed (frontend)  
**Target Platform**: Web (server: Linux, client: modern browsers ‚Äî Chrome, Firefox, Safari, Edge)  
**Performance Goals**: Thumbnails visible within 3 seconds of file selection (object URL, no upload); video upload ‚â§ 10 seconds for 100 MB on broadband (SC-004)  
**Constraints**: Max 5 images per submission (service-enforced); audio and video mutually exclusive (Pydantic + client); image MIME: JPEG/PNG/WEBP max 5 MB each; video MIME: MP4/WebM max 100 MB; client-side validation before any upload  
**Scale/Scope**: Single-tenant; feedback widget (authenticated users); admin dashboard (admin role)

## Implementation Phases

### Phase 1 ‚Äî Backend: Migration, Schemas, Service Extension

1. Write `010_extend_feedback_rich_media.sql`:
   - Create `feedback_images` table with `(feedback_id, display_order)` index and RLS
   - `ALTER TABLE feedback_submissions ADD COLUMN video_url TEXT`
   - Backfill: `INSERT INTO feedback_images SELECT id, image_url, 0 FROM feedback_submissions WHERE image_url IS NOT NULL`
   - `ALTER TABLE feedback_submissions DROP COLUMN image_url`
   - Update storage bucket MIME allowlist and size limit (100 MB) via Supabase config/seed

2. Update `app/schemas/feedback.py`:
   - `FeedbackImageResponse(id: UUID, storage_url: str, display_order: int)`
   - `FeedbackSubmitRequest`:
     - Remove `image_url: str | None`
     - Add `image_paths: list[str] | None` (max 5 validated in `@field_validator`)
     - Add `video_url: str | None`
     - Add `@model_validator` raising `422` if both `audio_url` and `video_url` are non-null
   - `FeedbackSubmitResponse`: replace `image_url` with `images: list[FeedbackImageResponse]`, add `video_url`
   - `FeedbackAdminItem`: same replacement

3. Extend `app/services/feedback/service.py`:
   - `upload_image(user_id, file)` ‚Üí validate MIME + size (‚â§ 5 MB), upload to Supabase Storage, return `storage_path`
   - `delete_upload(user_id, storage_path)` ‚Üí verify path ownership (`feedback/{user_id}/...`), delete from storage
   - `upload_video(user_id, file)` ‚Üí validate MIME (`video/mp4`, `video/webm`) + size (‚â§ 100 MB), upload, return `storage_path`
   - `submit_feedback()`:
     - After inserting `feedback_submissions` row, loop over `image_paths` and INSERT into `feedback_images` (check count ‚â§ 5 before loop)
     - Store `video_url` on submission row
   - `list_feedback()` (admin):
     - LEFT JOIN `feedback_images` ‚Üí aggregate into `images` array ordered by `display_order`
     - Add `video_url` to response
     - Generate signed URLs for all image paths and video_url
     - Remove `image_url` from response

4. Update `app/api/routes/feedback.py`:
   - `POST /api/feedback/upload/image` ‚Üí call `upload_image()`
   - `DELETE /api/feedback/upload/image` ‚Üí call `delete_upload()`, validate ownership
   - `POST /api/feedback/upload/video` ‚Üí call `upload_video()`
   - `DELETE /api/feedback/upload/video` ‚Üí call `delete_upload()`, validate ownership

5. Update `app/api/routes/admin.py`:
   - `GET /api/admin/feedback` response: use updated `FeedbackAdminItem` (images array, video_url)

### Phase 3 ‚Äî Frontend

1. Create `VideoRecorderService` in `video-recorder.service.ts`:
   - `MediaRecorder` + `MediaStream` lifecycle: `start()`, `stop()`, `cleanup()`
   - 2-minute cap via `setTimeout` ‚Üí auto-stop
   - Elapsed timer observable (RxJS `interval(1000)`)
   - Capability detection: `MediaRecorder.isTypeSupported('video/webm;codecs=vp8')` ‚Üí `canRecord$: Observable<boolean>`
   - On stop: collect `Blob` chunks, produce `objectUrl` for preview
   - MIME preference: `video/webm;codecs=vp8` ‚Üí fallback `video/mp4`

2. Update `FeedbackService` (`feedback.service.ts`):
   - `uploadImage(file: File): Observable<{ storage_path: string }>`
   - `deleteUpload(storage_path: string, type: 'image' | 'video'): Observable<void>`
   - `uploadVideo(file: File | Blob): Observable<{ storage_path: string }>`
   - Update `submitFeedback()` payload: `image_paths`, `video_url` instead of `image_url`

3. Update `FeedbackWidgetComponent`:
   - **Multi-image state**: `stagedImages: { file: File; objectUrl: string; storagePath?: string }[]` (max 5)
     - File input: `accept="image/jpeg,image/png,image/webp"`, `multiple`
     - On select: validate MIME + size client-side, push to `stagedImages`, render `objectUrl` thumbnail
     - Remove button per thumbnail: revoke object URL, call `deleteUpload()` if `storagePath` set, splice from array
     - Block 6th image: show inline "5 image limit reached" message
   - **Video state**: `stagedVideo: { blob?: Blob; file?: File; objectUrl?: string; storagePath?: string } | null`
     - Record button (hidden if `!canRecord$`): start `VideoRecorderService`, show elapsed timer
     - Stop button: stop recorder, set `objectUrl` preview
     - Re-record button: revoke object URL, call `deleteUpload()` if staged, reset `stagedVideo`
     - File upload input: `accept="video/mp4,video/webm"`, single file
     - On file select: validate MIME + size (‚â§ 100 MB) client-side, set `stagedVideo`
   - **Mutual exclusion**:
     - If `audio_url` is staged ‚Üí disable video section with message: "Remove audio to attach video"
     - If `stagedVideo` is set ‚Üí disable audio section with message: "Remove video to attach audio"
   - **On Submit**:
     - Upload images sequentially (each `uploadImage()` call); retry once on failure; collect `storage_path` values
     - Upload video (if staged) via `uploadVideo()`; retry once on failure; offer "Remove video" recovery
     - Build submit payload with `image_paths`, `audio_url`, `video_url`
   - **On abort / destroy**: call `deleteUpload()` for each `storagePath` in `stagedImages` and `stagedVideo` (if set)

4. Update `FeedbackAdminComponent`:
   - Expanded row: replace single image link with thumbnail grid (max 5 thumbnails); each opens lightbox / full-size view
   - Expanded row: video player (`<video controls>`) when `video_url` is present

5. Add i18n keys (`ar.json` + `en.json`):
   - `feedback.images.limit_reached`, `feedback.images.remove`, `feedback.images.attach`
   - `feedback.video.record`, `feedback.video.stop`, `feedback.video.re_record`, `feedback.video.upload`
   - `feedback.video.no_support`, `feedback.video.preview`
   - `feedback.media.audio_exclusive`, `feedback.media.video_exclusive`
   - `feedback.video.upload_failed`, `feedback.video.remove_and_submit`
